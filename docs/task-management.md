## 仕様書

### タスク完了時の手順

**重要**: 各タスクの完了時には、必ず以下の手順を実行してください。

1. タスクの実装・テストが完了したことを確認
2. 変更内容をステージングエリアに追加
3. コミットメッセージを作成してコミット
4. **リモートリポジトリにプッシュ**

```bash
git add .
git commit -m "完了: [タスクID] タスクタイトル"
git push origin main
```

### タスク管理ルール

- 依存タスクが完了してからタスクを開始する
- 複雑度8以上のタスクはサブタスクに分割することを推奨
- 各タスクの工数は日単位で見積もり
- ステータスは「未着手」「進行中」「完了」で管理
